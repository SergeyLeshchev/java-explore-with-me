Explore With Me — это бэкенд-приложение для организации событий. Пользователи могут создавать события, подавать заявки на участие, оставлять комментарии, а администраторы модерировать контент. Особенность системы интеграция сервиса статистики для отслеживания просмотров.

----

Технологический стек

Backend: Java 21, Spring Boot, Spring Data JPA (Hibernate), Spring Web

База данных: PostgreSQL (для основного сервиса и статистики)

Архитектура: Микросервисная (основной сервис + статистика)

Взаимодействие: HTTP/REST, RestTemplate

Тестирование: Postman

Контейнеризация: Docker, Docker Compose

----

Ключевые функции

Для пользователей:

Создание, редактирование и просмотр событий

Подача заявок на участие в событиях

Комментирование опубликованных событий

Отмена своих заявок

Поиск событий по фильтрам (категория, дата, текст, платность)

Для администраторов:

Модерация событий (публикация/отклонение)

Управление категориями событий

Создание подборок событий (compilations)

Управление пользователями

Просмотр статистики просмотров событий

----

Бизнес-логика:

Автоматическое подтверждение заявок при отключенной модерации или отсутствии лимита участников

Проверка пересечения временных интервалов событий

Валидация дат (событие нельзя создать менее чем за 2 часа)

Ограничение участников (лимиты и подтвержденные заявки)

Статусная модель событий (PENDING, PUBLISHED, CANCELED)

----

Архитектура и ключевые решения

Микросервисная архитектура:

Проект разделен на два независимых сервиса:

Основной сервис (ewm):

Вся бизнес-логика приложения

Управление событиями, пользователями, заявками, комментариями

Сложные SQL-запросы с фильтрацией и пагинацией

Работа с PostgreSQL через Spring Data JPA

Сервис статистики (stats):

Сбор статистики просмотров событий

Хранение данных о hits (URI, IP, время)

Предоставление агрегированной статистики (уникальные/все просмотры)

Модульная структура: server, client, dto

Модульность сервиса статистики:

stats-server: REST API, бизнес-логика, работа с БД

stats-client: HTTP-клиент для взаимодействия с основным сервисом

stats-dto: общие DTO для обоих модулей (исключает дублирование)
